# 2026-03-07 Fix Homebrew Cask install syntax error

## Context

Running `brew install cboone/tap/snappy` fails with a Ruby syntax error in the generated `Casks/snappy.rb` at line 28. The `postflight` block contains a raw shell command instead of proper Homebrew Cask Ruby DSL:

```ruby
# Generated (broken):
postflight do
  xattr -dr com.apple.quarantine "#{staged_path}/snappy"
end
```

The root cause is in `.goreleaser.yml` line 55. The `hooks.post.install` value uses raw shell syntax, but GoReleaser embeds it verbatim into the Cask file. It needs to use Homebrew's `system_command` DSL instead.

The secondary error (name conflict with homebrew/core `snappy`) only appears because the cask fails to load, causing Homebrew to fall back to formula resolution. Fixing the cask syntax resolves both errors.

Additionally, the tap repo contains an outdated `Formula/snappy.rb` (v0.3.1) from when GoReleaser used the now-deprecated `brews` section. This stale formula could cause future confusion but is in the separate `cboone/homebrew-tap` repo.

## Changes

### 1. Fix `.goreleaser.yml` hook syntax (lines 53-55)

Change:

```yaml
hooks:
  post:
    install: 'xattr -dr com.apple.quarantine "#{staged_path}/snappy"'
```

To:

```yaml
hooks:
  post:
    install: |
      system_command "/usr/bin/xattr", args: ["-dr", "com.apple.quarantine", "#{staged_path}/snappy"]
```

This matches the [GoReleaser Homebrew Casks documentation](https://goreleaser.com/customization/homebrew_casks/) for removing the quarantine bit.

### 2. Clean up stale formula from tap repo (separate repo)

Remove `Formula/snappy.rb` from `cboone/homebrew-tap` since snappy now ships as a cask, not a formula. This file is from v0.3.1 and is no longer generated by the current GoReleaser config. This step requires a commit to the tap repo.

## Files to modify

- `Formula/snappy.rb` in `cboone/homebrew-tap` (separate repo, delete)
